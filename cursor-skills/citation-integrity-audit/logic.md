# Audit Skeleton · 核查推进骨架

The advance order of a citation audit. The illustrations are
domain-neutral; the skeleton is field-independent — it holds for a proof, an
empirical study, a position piece, or a technical standard alike.
本文件给出引证核查的推进顺序。范例均为领域中性；骨架与具体领域无关——理论证明、
实证研究、立场论文或技术标准皆适用。

---

## Order of advance | 推进顺序（默认骨架）

1. **Close the citation set** — extract every in-text key and every bibliography
   entry, then intersect. An entry never cited (orphan) and a key cited but
   absent (dangling) are defects detectable before a single source is opened —
   close the universe before judging it.
   **闭合引用集**——抽出正文每个引用键与文献表每个条目，求交。从未被引用的条目
   （孤儿）与被引却缺失的键（悬空）是开任何来源之前就能发现的缺陷——先把全集闭合，
   再评判。
   - e.g. *27 keys in the body, 27 entries in the list, intersection complete:
     no orphan, no dangling.*
     范例："正文 27 键、文献表 27 条，交集完整：无孤儿、无悬空。"

2. **Locate the pair** — for each key, find the exact sentence it is attached to.
   The audit unit is (claim, source), never the reference alone: a reference
   severed from the claim it serves cannot be judged for relevance.
   **定位"对"**——为每个键找到它所挂的那一句。核查单元是"（主张，来源）"，绝非
   孤立引用：与所服务主张切断的引用，无从判断其相关性。

3. **Resolve existence against a primary record** — confirm the work exists with
   the stated author, venue, year, and pages, against a primary or authoritative
   record (DOI, arXiv, publisher, official spec). What cannot be resolved fails
   here; its content is not yet in question.
   **以一手记录解析存在性**——对照一手或权威记录（DOI、arXiv、出版商、官方规范），
   确认作品存在且作者/出处/年份/页码相符。无法解析者在此判败；其内容尚不进入讨论。

4. **Extract the load-bearing sentence** — from the resolved source, pull the one
   sentence that does the supporting. If no sentence in the source bears the
   claim, the citation is decorative or misattributed — and that is the finding,
   not a dead end.
   **抽取承重句**——从已解析的来源中，抽出真正承载支撑的那一句。若来源中无任何
   句子承载该主张，则引用为装饰性或张冠李戴——这本身就是结论，而非死路。

5. **Confront, verbatim and bilingual** — set the source sentence beside the
   paper's own wording; quote both, summarize neither. The match becomes something
   the reader sees rather than something the audit asserts. For a bilingual paper,
   carry both language editions.
   **逐字且双语地对质**——把被引原句与论文自身措辞并置；两者皆逐字引用，皆不转述。
   吻合于是成为读者"看到"之物，而非核查"宣称"之物。论文若有双语稿，则两版并载。

6. **Test placement** — ask whether the citation sits under the claim that needs
   it. A real, well-matched source filed under the wrong sentence is a defect of
   placement — a distinct axis from existence and from match. Keep the three
   apart: a source can pass two and fail the third.
   **检验落点**——追问引用是否落在需要它的那条主张下。真实且吻合的来源若挂错句子，
   即"落点"缺陷——一条与"存在""吻合"相区别的独立轴。三轴须分开：一条来源可过其二、
   败其一。
   - Anti-pattern (do not collapse): a single verdict that fuses "is it real / does
     it say it / is it in the right place", hiding which axis failed.
     反模式（勿合并）：用单一判定把"是否真实／是否如是说／是否落对位置"熔在一起，
     掩盖究竟哪条轴失败。

7. **Attach the link** — every row carries a resolvable pointer to the primary
   source. This is not decoration: the audit's own credibility rests on being
   re-checkable — the same discipline it imposes on the paper. An audit that
   cannot be re-checked is just another assertion.
   **附上链接**——每行附一手来源的可解析地址。这非装饰：核查自身的可信度，正建立
   在"可被复核"之上——这恰是它施于论文的同一条纪律。无法被复核的核查，不过是又一
   句断言。

8. **Grade, then state the residue** — assign one verdict from a fixed lexical
   set (match / true-but-misplaced / match-but-non-primary), write a named note
   for every nuance, and close with what stays unresolved. The audit earns trust
   by marking its own soft spots, not by clearing every row.
   **定级，再陈残差**——给一个取自固定词面集的判定（吻合／真实但落点偏移／吻合但
   来源非一手），每处微妙之处写一条点名备注，并以"尚未解决者"收束。核查靠标出自身
   软肋赢得信任，而非靠让每一行都过关。
   - e.g. *one row true but mis-placed (flag it); one row matched but the source
     is a non-peer-reviewed project doc (label it); the rest a plain match.*
     范例："一行真实但落点错（标出）；一行吻合但来源为未经同行评审的项目文档
     （标注）；其余记'吻合'。"

## Self-check | 自检清单

- [ ] Is the citation set closed — every key matched to an entry, every entry
      cited, orphans and dangling keys named? |
      引用集是否闭合——每键对应一条、每条被引、孤儿与悬空均点名？
- [ ] Is each verdict pinned to the exact sentence the citation supports, not to
      the reference in isolation? |
      每项判定是否钉在引用支撑的那一句上，而非孤立的引用？
- [ ] Is every work resolved to a primary or authoritative record before its
      content is judged? |
      每件作品是否在判其内容前已解析到一手或权威记录？
- [ ] Is the evidence a verbatim source sentence, never a paraphrase? |
      证据是否为逐字被引原句，而非转述？
- [ ] Is the source sentence set against the paper's own wording, bilingually? |
      被引原句是否与论文自身措辞双语并置？
- [ ] Are existence, content-match, and placement kept as three separate axes? |
      存在、内容吻合、落点是否保持为三条独立轴？
- [ ] Does every row carry a resolvable link to the primary source? |
      每行是否都附一手来源的可解析链接？
- [ ] Does the audit close by naming what stays unresolved, rather than clearing
      every row? | 核查是否以"点名尚未解决者"收束，而非让每行都过关？
